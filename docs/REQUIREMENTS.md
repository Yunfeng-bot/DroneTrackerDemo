# 无人机伴飞单目标追踪系统 (Drone Tracker System)
## 需求说明书 (PRD)

### 1. 项目背景
当前的无人机航拍与特种监控场景中，亟需一种能够“零干预、零样本”实现单目标死咬追踪的视觉系统。传统的基于 YOLO 等多目标追踪方案（Tracking-by-Detection）需要耗费大量成本收集特定物种的数据集进行重新训练和量化，极度拖慢开发周期且无法应对未知目标（如临时指派的某一辆特殊的工程车、一条船甚至一个人）。
本项目旨在开发一套**“提前给定图片（模板），立刻死锁伴飞”**的高性能单目标追踪（SOT）平台，并制定了从快速演示到工业级落地的完整两步走路线。

### 2. 产品阶段划分与演进路线

#### Phase 1: 概念验证演示 MVP (当前版本)
*   **平台载体：** 现有的通用 Android 智能手机或平板电脑。
*   **核心痛点解决：** 以几乎为零的硬件打样成本，极速验证计算机视觉追踪算法输出的数据，能否平稳指挥无人机云台/飞控系统（验证 PID 闭环逻辑）。
*   **核心要求：** 必须在一周内开发完毕，能够拿给投资人或大老板在办公室进行实机“画框追踪”的 Demo 演示。不需要解决高难度遮挡跟丢问题，只需要轻量级运行，保证高帧率（>30fps）。

#### Phase 2: 工业级量产落地 (远期目标)
*   **平台载体：** 挂载在无人机上的轻量级边缘计算盒子（基于瑞芯微 RK3588 或类似带有独立 NPU 的 Linux 芯片板卡）。
*   **核心痛点解决：** 解决 Android 设备长时间高负载发热降频问题，并解决目标在复杂环境中（变形、反光、短暂遮挡）的跟丢问题。
*   **核心要求：** 无需 UI 图形界面。系统开机自启，接收自无人机的真实网络压缩视频流（H.264/H.265），利用深度学习孪生网络（Siamese Network）进行基于特征 DNA 的军工级死锁追踪，输出坐标点指令。

### 3. 功能需求细则 (Phase 1)
1. **纯净流摄取：** 不使用传统卡顿的 Camera API，必须应用最新的 CameraX 引擎低延迟抽取物理镜头的裸流（无压缩画面）。
2. **通缉令下达 (ROI Selection)：** 在屏幕提供一层透明可手势滑动的 UI。支持飞手用手动画一个方框，框住任何他不认识的物体。
3. **坐标实时输出：** 松手的瞬间，算法接管。在后续任何一帧画面流中，系统必须实时（<30ms/帧）吐出该物体当前在画面内的最新 (X, Y) 坐标中心点和边框大小。
4. **视觉反馈：** 在屏幕上使用醒目的有色方框实时重绘追踪目标，供人类肉眼验证。
5. **系统重置：** 提供一键重置按钮，随时打断当前的追猎过程，重新选取新的目标。

### 4. 非功能性与性能需求
*   **零样本 (Zero-Shot)：** 系统绝对禁止使用任何预设了实体标签（“车”、“人”、“狗”）的目标检测库。必须做到“框什么追什么”。
*   **延迟与流畅度 (FPS)：** 纯算法处理耗时在普通移动端 CPU 上不可超过 33ms（保证最底线 30 FPS 输出），否则飞控系统的 PID 调节会出现灾难性的抖动（画龙）。
*   **无网络依赖：** 所有的视觉计算必须离线、纯本地边缘端完成（不可将视频传给云端服务器计算，图传云端延迟太高必炸机）。
